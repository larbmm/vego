### 1. 配置文件格式 ✅


### 2. 配置加载代码 ✅

文件：`src/config/config.ts`

在 `loadConfig()` 函数中添加了读取 `display_name` 的代码：

```typescript
// Load character configs
for (const [charName, charConfig] of Object.entries(rawConfig.character || {})) {
  const charData = charConfig as any;
  config.character[charName] = {
    name: charName,
    display_name: charData.display_name,  // ← 新增这一行
    path: charData.path,
    telegram_bot_token: charData.telegram_bot_token,
    discord_bot_token: charData.discord_bot_token,
    feishu_app_id: charData.feishu_app_id,
    feishu_app_secret: charData.feishu_app_secret,
  };
}
```

已运行 `npm run build`，TypeScript 代码已编译为 JavaScript。

## 如果重启后仍然显示英文

### 可能原因 1：浏览器缓存

**解决方案：**

- Mac Chrome/Safari：按 `Cmd + Shift + R` 强制刷新
- 或者清除浏览器缓存后重新打开

### 可能原因 2：应用未完全重启

**解决方案：**

```bash
# 确保完全停止所有进程
pm2 delete vego
pm2 start ecosystem.config.js

# 或者
pkill -f "node.*vego"
npm start
```

### 可能原因 3：使用了错误的配置文件

**检查方法：**

查看应用启动时的日志，应该看到：

```
[Config] Using .vego directory: /Users/你的用户名/.vego
```

或者：

```
[Config] Using .vego directory: /path/to/project/.vego
```

确保这个目录下的 `config.toml` 文件包含 `display_name` 字段。

## 技术说明

### 为什么需要重启？

Node.js 应用在启动时会加载配置文件到内存中。虽然我们修改了：

1. 配置文件（`.vego/config.toml`）
2. 配置加载代码（`src/config/config.ts`）
3. 编译后的代码（`build/config/config.js`）

但是**正在运行的进程仍然使用旧的内存中的配置**。只有重启应用，才会重新读取配置文件并加载新的代码。

### 修复的完整流程

1. ✅ 更新配置文件格式（从注释改为 `display_name = "..."` 格式）
2. ✅ 修改 TypeScript 代码，添加读取 `display_name` 的逻辑
3. ✅ 编译 TypeScript 代码为 JavaScript（`npm run build`）
4. ⏳ **重启应用**（这一步需要你来做）
5. ✅ 浏览器访问 Web 界面，看到中文名称
