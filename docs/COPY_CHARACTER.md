# 复制角色功能说明

## 功能概述

从现有角色复制创建新角色，快速获得完整的人设文件，只需修改部分内容即可。

## 使用场景

### 1. 创建角色变体
基于一个成熟的角色创建不同版本：
- **芊芊** → **芊芊（工作版）**：调整为职场风格
- **曦月** → **曦月（学生版）**：调整为校园风格
- **助手** → **助手Pro**：增强专业能力

### 2. 创建相似角色
创建性格相似但细节不同的角色：
- **小红** → **小蓝**：保持性格，改变背景故事
- **Alice** → **Bob**：保持说话风格，改变性别和身份

### 3. 快速测试
基于现有角色快速创建测试角色：
- 测试不同的人设组合
- 测试新的对话风格
- 测试特定场景的反应

## 使用方法

### 通过Web界面

1. 点击"角色管理" → "+ 新增角色"
2. 填写新角色信息：
   - **英文ID**：`alice_v2`
   - **中文名称**：`爱丽丝2.0`
   - **复制自**：选择 `alice`
3. 点击"创建角色"
4. 系统自动复制人设文件
5. 点击"编辑"检查并修改人设

### 复制过程

系统会自动：
1. 复制源角色的所有 `.md` 人设文件
2. 替换文件中的角色名称
   - 英文ID：`alice` → `alice_v2`
   - 中文名称：`爱丽丝` → `爱丽丝2.0`
3. 创建新的数据库文件（空的）
4. 在配置文件中添加新角色

## 示例场景

   - 说话风格：改为"先生"称呼，减少亲昵语气

```
基本信息：18岁女生，大二学生
性格：稳定、有点倔强
关系：伴侣
```

**新角色：晨阳（男性）**
1. 复制自"曦月"
2. 修改人设：
   - 基本信息：改为男性，调整身高等
   - 性格：保持核心特点
   - 说话风格：调整为男性化表达

### 示例3：创建测试角色

**原角色：小明（成熟版）**
```
完整的人设，经过长期调教
```

**新角色：小明_test**
1. 复制自"小明"
2. 用于测试：
   - 测试新的对话策略
   - 测试不同的回复风格
   - 不影响原角色的数据

## 复制后的修改建议

### 必须修改的内容

1. **basic.md - 基本信息**
   - 名字（已自动替换）
   - 年龄、性别、身高等个人信息
   - 与用户的关系

2. **background.md - 背景故事**
   - 成长经历
   - 重要事件
   - 与用户的相识过程

### 可选修改的内容

3. **personality.md - 性格特点**
   - 如果要创建不同性格的变体，需要修改
   - 如果只是换个名字，可以保持不变

4. **speaking.md - 说话风格**
   - 口头禅、常用语
   - 称呼方式
   - 语气特点

5. **rules.md - 行为规则**
   - 通常可以保持不变
   - 除非要改变角色的核心行为模式

6. **interests.md - 兴趣爱好**
   - 根据新角色的设定调整

## 注意事项

### 1. 名称替换

系统会自动替换：
- 英文ID：`qianqian` → `qianqian_v2`
- 中文名称：`芊芊` → `芊芊2.0`

但可能不完美，建议：
- 创建后检查所有人设文件
- 手动修正未替换的地方
- 特别注意背景故事中的名字

### 2. 数据独立

- 新角色有独立的数据库
- 不会共享对话历史
- 不会影响原角色

### 3. 配置文件

新角色会添加到配置文件：
```toml
[character.alice_v2]
display_name = "爱丽丝2.0"
path = "workspace_alice_v2"
# telegram_bot_token = ""  # 需要单独配置
```

需要为新角色配置独立的 bot token。

### 4. 重启应用

创建后需要重启应用才能使用新角色。

## 与手动复制的区别

### Web界面复制（推荐）

**优点：**
- 自动替换角色名称
- 自动更新配置文件
- 自动创建数据库
- 界面操作简单

**缺点：**
- 只复制人设文件，不复制对话历史

### 手动复制

**优点：**
- 可以复制对话历史
- 可以选择性复制文件
- 更灵活

**缺点：**
- 需要手动修改所有名称
- 需要手动更新配置文件
- 容易出错

## 常见问题

### Q: 复制后的角色会共享对话历史吗？
A: 不会。每个角色有独立的数据库，对话历史完全独立。

### Q: 可以复制对话历史吗？
A: Web界面不支持。如果需要，可以手动复制数据库文件：
```bash
cp .vego/workspace_alice/memory.db .vego/workspace_alice_v2/memory.db
```

### Q: 复制后修改原角色会影响新角色吗？
A: 不会。复制后两个角色完全独立。

### Q: 可以从新角色复制回原角色吗？
A: 可以，但需要手动操作。建议使用版本控制（git）管理人设文件。

### Q: 复制时会复制bot token吗？
A: 不会。新角色需要单独配置bot token。

### Q: 名称替换不完整怎么办？
A: 创建后点击"编辑"，手动检查并修改所有人设文件。

### Q: 可以一次复制多个角色吗？
A: 不可以。需要逐个创建。

### Q: 复制失败怎么办？
A: 检查：
1. 源角色是否存在
2. 源角色的人设文件是否完整
3. 新角色名称是否符合规范
4. 查看系统日志了解错误原因

## 最佳实践

### 1. 命名规范

使用清晰的命名区分版本：
- `alice` → `alice_v2`、`alice_work`、`alice_casual`
- `qianqian` → `qianqian_pro`、`qianqian_test`

### 2. 版本管理

为重要角色创建备份：
```bash
# 创建备份版本
# 在Web界面：复制自 qianqian，命名为 qianqian_backup
```

### 3. 测试流程

1. 从成熟角色复制创建测试版本
2. 在测试版本上尝试新的人设
3. 测试满意后，应用到正式版本

### 4. 文档记录

在 `index.md` 中记录：
- 基于哪个角色创建
- 主要修改了什么
- 创建目的

```markdown
# 爱丽丝2.0

角色工作区

创建时间：2024-01-01
基于角色：alice
主要修改：调整为职场风格，更专业的说话方式
```

## 技术细节

### 复制过程

1. 验证源角色存在
2. 创建新角色目录结构
3. 遍历源角色的 `persona` 目录
4. 复制所有 `.md` 文件
5. 在文件内容中替换名称：
   - 英文ID（不区分大小写）
   - 中文显示名称（精确匹配）
6. 创建空数据库
7. 更新配置文件
8. 重新加载配置

### 名称替换逻辑

```typescript
// 替换英文ID（不区分大小写）
content = content.replace(new RegExp(copy_from, 'gi'), name);

// 替换中文名称（精确匹配）
if (sourceDisplayName && targetDisplayName) {
  content = content.replace(
    new RegExp(sourceDisplayName, 'g'), 
    targetDisplayName
  );
}
```

### 不复制的内容

- 对话历史（`memory.db`）
- 记忆文件（`memory/` 目录）
- 关系文件（`relationship/` 目录）
- 调度器状态（`scheduler_state.json`）

只复制 `persona/` 目录下的人设文件。
