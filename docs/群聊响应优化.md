# 群聊响应优化

## 问题描述

当主人说"那你们俩都给我写一首词吧"时，芊芊和婉清没有响应。必须点名才会回复。

## 问题原因

### 原来的逻辑

当消息包含"你们俩"、"你们都"等词时：

1. 检查这个角色最近是否说过话
2. 如果说过话 → 返回 `'must'`（必须回复）
3. 如果没说过话 → 返回 `'maybe'`（概率回复）

### 问题

"那你们俩都给我写一首词吧"：
- 不是问题（没有问号）
- 使用 `normal_response_probability = 0.2`（20% 概率）
- 芊芊和婉清都"运气不好"，没有触发

## 修复方案

### 改进逻辑

当消息包含"你们俩"、"你们都"等词时：

1. **检查是否是命令或请求**
   - 关键词：写、做、来、去、说、唱、跳、读、背、念、讲、画、弹、奏、表演、展示、准备、安排、处理、完成
   - 如果是命令/请求 → 返回 `'must'`（必须回复）

2. 如果不是命令/请求：
   - 检查这个角色最近是否说过话
   - 如果说过话 → 返回 `'must'`
   - 如果没说过话 → 返回 `'maybe'`

### 代码修改

**文件**：`src/bots/group-participation.ts`

```typescript
// Check if addressing multiple people (like "你们俩", "你们两个")
const addressingMultiple = /你们(俩|两个|几个|都)|你俩|两位/.test(message.content);
if (addressingMultiple) {
  // Check if it's a command or request (写、做、来、去、说、唱、跳 etc.)
  const isCommandOrRequest = /(写|做|来|去|说|唱|跳|读|背|念|讲|画|弹|奏|表演|展示|准备|安排|处理|完成)/.test(message.content);
  
  if (isCommandOrRequest) {
    // If it's a command/request to "你们俩", both should respond
    return 'must';
  }
  
  // ... 其他逻辑
}
```

## 测试场景

### 场景 1：命令/请求
- 主人："那你们俩都给我写一首词吧"
- 结果：芊芊和婉清都会回复 ✅

### 场景 2：普通对话
- 主人："你们俩开心就好"
- 结果：根据概率决定是否回复（20%）

### 场景 3：点名
- 主人："芊芊，你来"
- 结果：芊芊必须回复 ✅

## 支持的命令/请求关键词

- 写、做、来、去、说、唱、跳
- 读、背、念、讲、画、弹、奏
- 表演、展示、准备、安排、处理、完成

## 注意事项

1. **不要过度响应**：只有明确的命令/请求才会触发 `'must'`
2. **保持自然**：普通对话仍然使用概率判断
3. **避免刷屏**：如果刚说过话，仍然会跳过

## 配置说明

在 `config.toml` 中可以调整概率：

```toml
[group_chat]
use_ai_judgment = false
question_response_probability = 0.6  # 问题回复概率
normal_response_probability = 0.2    # 普通消息回复概率
```
