# 配置文件结构优化

## 改进内容

将 `config.toml` 文件重新组织，**所有角色配置 `[character.*]` 统一放在文件最后**。

## 优化后的结构

```toml
[api]
# API 配置

[memory]
# 内存配置

[scheduler]
# 调度器配置

[weekly_review]
# 周回顾配置

[proactive_chat]
# 主动聊天配置

[group_chat]
# 群聊配置

# ============================================
# 角色配置 (Characters Configuration)
# ============================================

[character.xiyue]
# 曦月的配置

[character.qianqian]
# 芊芊的配置

[character.wanqing]
# 婉清的配置

[character.xiaoling]
# 小玲的配置
```

## 优点

1. **结构清晰**：系统配置在前，角色配置在后，一目了然
2. **易于管理**：所有角色集中在一起，方便查看和编辑
3. **便于扩展**：添加新角色时直接追加到文件末尾即可
4. **符合习惯**：配置文件通常将可变内容（角色）放在最后

## 代码兼容性

### 创建角色

现有代码使用**追加方式**添加新角色：

```typescript
configContent += newCharacterConfig;
```

这种方式天然地将新角色添加到文件末尾，完美符合新结构。✅

### 删除角色

使用正则表达式删除角色配置：

```typescript
const sectionRegex = new RegExp(`\\n*\\[character\\.${character}\\][\\s\\S]*?(?=\\n\\[|$)`, 'g');
configContent = configContent.replace(sectionRegex, '');
```

这个正则会匹配从 `[character.xxx]` 到下一个 `[` 或文件末尾的所有内容，无论角色在文件的什么位置都能正确删除。✅

### 配置加载

配置加载使用 TOML 解析器，不关心配置项的顺序，完全兼容。✅

## 使用说明

### 手动添加角色

如果你手动编辑 `config.toml` 添加角色，请将新角色配置添加到文件末尾：

```toml
# ... 其他配置 ...

# ============================================
# 角色配置 (Characters Configuration)
# ============================================

[character.existing1]
# ...

[character.existing2]
# ...

[character.new_character]  # ← 新角色添加在这里
display_name = "新角色"
path = "workspace_new_character"
telegram_bot_token = "..."
```

### 通过 Web 界面添加

Web 界面会自动将新角色追加到文件末尾，无需手动操作。

### 删除角色

无论通过 Web 界面还是手动删除，都不会影响文件结构。

## 迁移说明

如果你有旧的配置文件（角色配置在前面），可以：

1. **自动迁移**：重启应用后，下次创建/删除角色时会自动调整
2. **手动迁移**：参考 `.vego/config.toml` 的新结构，手动重新组织

或者直接使用已更新的 `.vego/config.toml` 文件（已经重新组织好了）。

## 注意事项

1. **备份自动创建**：每次修改配置文件都会自动创建备份（`.backup.时间戳`）
2. **重启生效**：修改配置文件后需要重启应用才能生效
3. **TOML 语法**：确保手动编辑时遵循 TOML 语法规则
