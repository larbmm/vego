# 创建新角色指南

## VEGO_HOME 目录说明

系统会按以下优先级查找 `.vego` 目录：

1. **~/.vego** （用户主目录，优先级最高）
2. **当前工作目录/.vego** （运行程序的目录）
3. **项目根目录/.vego** （代码所在目录）

所有角色数据、配置文件都会存储在找到的第一个 `.vego` 目录中。

**建议：** 如果你想让数据独立于项目目录，可以在用户主目录创建 `~/.vego` 目录并将配置文件移动过去。

## 通过Web界面创建

### 步骤1：打开角色管理

1. 访问 Web 管理界面（默认：http://localhost:3000）
2. 点击左侧菜单的"角色管理"
3. 点击右上角的"+ 新增角色"按钮

### 步骤2：输入角色信息

**英文ID（必填）：**
- 只能使用**小写字母**和**下划线**
- 例如：`alice`、`xiao_ming`、`my_character`
- 用于系统内部标识和文件夹名称
- 不能使用中文、大写字母、空格或特殊字符

**中文名称（可选）：**
- 可以使用任何字符，包括中文、数字、符号
- 例如：`爱丽丝`、`小明`、`我的角色`
- 用于界面显示，让用户更容易识别
- 不填则使用英文ID作为显示名称

**复制自（可选）：**
- 从下拉框中选择一个现有角色
- 系统会复制该角色的所有人设文件
- 自动替换文件中的角色名称
- 适合创建相似的角色，只需修改部分内容

### 步骤3：系统自动创建

系统会自动创建：
- ✅ 角色工作区目录（`.vego/workspace_[角色名]`）
- ✅ 完整的人设模板文件（6个markdown文件）
  - 如果选择了"复制自"，则复制现有角色的人设
  - 否则使用默认模板
- ✅ 数据库文件（`memory.db`）
- ✅ 配置文件条目（`config.toml`）

### 步骤4：编辑人设

1. 在"角色管理"页面找到新创建的角色
2. 点击"编辑"按钮
3. 依次编辑以下文件：
   - **basic.md** - 基本信息（名字、年龄、身份、关系等）
   - **personality.md** - 性格特点（性格底色、情绪节奏、反应方式）
   - **background.md** - 背景故事（成长经历、重要事件）
   - **speaking.md** - 说话风格（语气、口头禅、时段调整）
   - **rules.md** - 行为规则（身份认知、核心原则、避免的行为）
   - **interests.md** - 兴趣爱好（喜欢的事物、特长、梦想）

### 步骤5：配置Bot Token

1. 点击左侧菜单的"配置管理"
2. 找到新角色的配置段落：
   ```toml
   [character.角色ID]
   display_name = "角色中文名"
   path = "workspace_角色ID"
   # telegram_bot_token = ""
   # discord_bot_token = ""
   ```
3. 取消注释并填入你的 Bot Token：
   ```toml
   [character.alice]
   display_name = "爱丽丝"
   path = "workspace_alice"
   telegram_bot_token = "你的Telegram Bot Token"
   # discord_bot_token = "你的Discord Bot Token"
   ```
4. 点击"保存配置"

### 步骤6：重启应用

配置保存后，需要重启应用才能生效：

```bash
# 停止应用（Ctrl+C）
# 重新启动
npm start
```

## 人设模板说明

### basic.md - 基本信息
定义角色的基础属性，包括名字、年龄、身份、外貌、与用户的关系等。

### personality.md - 性格特点
描述角色的性格底色、情绪节奏、遇到不同情况时的反应方式。这是让角色"活起来"的关键。

### background.md - 背景故事
角色的成长经历、重要事件、与用户的相识过程等。背景故事让角色更有深度。

### speaking.md - 说话风格
定义角色的语气、常用语、口头禅、不同时段的说话方式等。这决定了角色的"声音"。

### rules.md - 行为规则
最重要的文件！定义角色的身份认知、核心原则、避免的行为。确保角色始终保持真实感。

### interests.md - 兴趣爱好
角色喜欢和不喜欢的事物、特长、梦想等。让角色更加立体。

## 注意事项

1. **角色ID与显示名称**
   - **英文ID**：用于系统内部、文件夹名称、配置文件
     - ✅ 正确：`alice`、`xiao_ming`、`my_bot`
     - ❌ 错误：`Alice`、`小明`、`my-bot`、`my bot`
   - **中文名称**：用于界面显示，可以使用任何字符
     - ✅ 正确：`爱丽丝`、`小明`、`我的机器人`、`Alice`
     - 可选字段，不填则使用英文ID

2. **配置文件格式**
   ```toml
   [character.英文ID]
   display_name = "中文名称"  # 可选
   path = "workspace_英文ID"
   telegram_bot_token = "..."
   ```

2. **人设编写建议**
   - 尽量详细，细节越多角色越真实
   - 使用具体的例子而不是抽象的描述
   - 考虑角色的不完美之处，完美的角色不真实
   - 定义角色的边界和底线

3. **配置Bot Token**
   - Telegram Bot Token 从 [@BotFather](https://t.me/BotFather) 获取
   - Discord Bot Token 从 [Discord Developer Portal](https://discord.com/developers/applications) 获取
   - 不要泄露你的 Token！

4. **测试角色**
   - 创建后先在私聊中测试
   - 观察角色的回复是否符合人设
   - 根据测试结果调整人设文件
   - 可以在"对话管理"中查看和编辑历史消息

## 常见问题

### Q: 角色文件创建在哪里？
A: 系统会按优先级查找 `.vego` 目录：
1. `~/.vego` （用户主目录）
2. 当前工作目录的 `.vego`
3. 项目根目录的 `.vego`

角色会创建在找到的第一个目录中。启动应用时会在日志中显示使用的路径：
```
[Config] Using .vego directory: /path/to/.vego
```

### Q: 创建角色后在列表中看不到？
A: 需要重启应用才能加载新角色。

### Q: 角色回复不符合人设？
A: 检查 `rules.md` 文件，确保核心原则清晰明确。可以在"对话管理"中查看历史对话，分析问题。

### Q: 可以复制现有角色的人设吗？
A: 可以！在文件管理器中复制 `.vego/workspace_现有角色/persona/` 目录下的文件到新角色的 persona 目录，然后修改即可。

### Q: 如何删除角色？
A: 在"角色管理"页面点击"删除"按钮，然后在"配置管理"中删除对应的配置段落，最后重启应用。

## 高级技巧

### 1. 从现有角色复制创建

**使用场景：**
- 创建相似的角色（如不同性格的助手）
- 基于成熟的人设创建变体
- 快速创建测试角色

**操作方法：**
1. 在创建角色时，从"复制自"下拉框选择一个现有角色
2. 系统会复制该角色的所有人设文件
3. 自动替换文件中的角色名称
4. 创建后检查并修改需要调整的部分

**示例：**
- 基于"芊芊"创建"芊芊2号"，调整性格更活泼
- 基于"曦月"创建"曦月（工作版）"，调整为职场风格

### 2. 手动复制角色工作区
### 2. 手动复制角色工作区

如果你已经有一个调教得很好的角色，可以手动复制：

```bash
# 复制整个工作区
cp -r .vego/workspace_xiyue .vego/workspace_newchar

# 修改人设文件
# 在配置文件中添加新角色配置
```

### 3. 多角色共享部分人设
可以创建共享的人设片段，在多个角色中引用：

```markdown
# rules.md
<!-- 引用共享规则 -->
{{include: ../shared/common_rules.md}}

# 角色特定规则
...
```

### 4. 版本控制
建议使用 Git 管理人设文件：

```bash
cd .vego/workspace_mychar
git init
git add .
git commit -m "Initial character setup"
```

这样可以追踪人设的修改历史，必要时回滚。
