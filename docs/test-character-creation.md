# 测试角色创建功能

## 修复的问题

### 问题1：创建后看不到角色
**原因：** config 对象在应用启动时加载，创建角色后虽然更新了配置文件，但内存中的 config 没有更新。

**修复：**
1. 添加 `reloadConfig()` 函数重新加载配置
2. 创建角色后调用 `reloadConfig()` 更新内存中的 config
3. 删除角色后也调用 `reloadConfig()`

### 问题2：第二次创建提示"已存在"
**原因：** 第一次创建后配置文件已更新，但内存中的 config 没更新，所以检查时认为不存在。第二次创建时，配置文件已经有了这个角色，所以提示已存在。

**修复：** 同上，通过 `reloadConfig()` 保持内存和文件同步。

### 问题3：角色创建在错误的目录
**原因：** 创建角色时使用了 `process.cwd()` 而不是配置系统中的 `VEGO_HOME`。

**修复：**
1. 导出 `getVegoHome()` 函数
2. 创建角色时使用 `getVegoHome()` 获取正确的路径
3. 确保角色创建在配置系统识别的目录中

1. 启动应用：`npm start`
2. 打开 Web 界面：http://localhost:3000
3. 点击"角色管理" → "+ 新增角色"
4. 输入角色名称：`test_alice`
5. 点击"创建角色"

**预期结果：**
- ✅ 显示成功提示
- ✅ 角色列表中立即显示新角色
- ✅ 可以点击"编辑"查看人设模板
- ✅ 配置文件中添加了角色配置

### 测试2：连续创建多个角色

1. 创建第一个角色：`test_bob`
2. 创建第二个角色：`test_charlie`
3. 创建第三个角色：`test_david`

**预期结果：**
- ✅ 每个角色都能成功创建
- ✅ 不会提示"已存在"
- ✅ 角色列表实时更新

**预期结果：**

## 路径验证

启动应用时，查看日志中的路径：
```
[Config] Using .vego directory: /path/to/.vego
```

创建角色后，验证工作区是否在这个目录下：
```bash
ls /path/to/.vego/workspace_*
```

## 注意事项

1. **角色名称规则：** 只能使用小写字母和下划线
2. **配置生效：** 虽然列表会立即更新，但要让角色真正运行，仍需重启应用
3. **备份：** 每次修改配置文件都会自动备份
4. **日志：** 创建/删除操作会记录到系统日志中
