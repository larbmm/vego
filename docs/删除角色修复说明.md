# 删除角色功能修复

## 问题描述

删除角色时出现错误：

```
删除失败：EBUSY: resource busy or locked, unlink 'D:\code\vego\.vego\workspace_xiaoling\memory.db'
```

## 问题原因

SQLite 数据库文件在被应用使用时会被锁定。删除角色时，如果该角色的 Character 实例还在内存中运行，数据库连接没有关闭，就无法删除数据库文件。

## 修复方案

在删除角色目录之前，先关闭数据库连接：

1. 从应用实例中获取该角色的 Character 对象
2. 调用 `char.close()` 关闭数据库连接
3. 然后再删除文件和目录

## 修改的代码

文件：`src/web/server.ts`

在删除角色的 API 中添加了关闭数据库的逻辑：

```typescript
// Close database connection if character is loaded in app
const { getAppInstance } = await import('../index.js');
const appInstance = getAppInstance();

if (appInstance) {
  const char = appInstance.getCharacter(character);
  if (char) {
    // Close database connection to unlock the file
    char.close();
    WebLogger.info(`Closed database connection for character: ${character}`);
  }
}
```

## 注意事项

1. **删除操作不可恢复**：删除角色会永久删除所有数据（人设、对话历史、数据库）
2. **自动备份配置**：删除时会自动备份 config.toml 文件
3. **需要重启应用**：删除后建议重启应用，确保内存中的数据完全清理

## 技术细节

### 为什么会锁定？

SQLite 数据库使用文件锁来保证数据一致性。当数据库连接打开时：

- Windows 系统会锁定 `.db` 文件
- 可能还会锁定 `.db-shm` 和 `.db-wal` 文件（WAL 模式）

### Character.close() 做了什么？

```typescript
close(): void {
  if (this.memoryManager) {
    this.memoryManager.close();
  }
}
```

`memoryManager.close()` 会关闭 SQLite 数据库连接，释放文件锁。

### 为什么不直接重启应用？

虽然重启应用也能释放锁，但：

1. 用户体验不好（需要手动重启）
2. 其他角色的服务会中断
3. 不符合"热删除"的设计理念

通过代码关闭连接，可以实现无需重启的删除功能。

## 相关问题

### Q: 删除后角色还在列表中？

A: 刷新浏览器页面（Ctrl+Shift+R）

### Q: 删除后配置文件没更新？

A: 检查 `.vego/config.toml`，应该已经移除了该角色的配置段。如果没有，查看备份文件。

### Q: 删除后还能恢复吗？

A: 不能。删除是永久性的。如果需要保留数据，建议先备份 workspace 目录。
