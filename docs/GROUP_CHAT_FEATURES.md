# 群聊功能说明

## 已实现的功能

### 1. 场景区分 ✅

系统会自动识别私聊和群聊场景，并给AI不同的行为指导：

#### 私聊场景
- AI可以更加亲密和放松
- 可以说一些私密的话
- 可以表达更多的情感
- 这是两个人的私密空间

#### 群聊场景
- 保持得体和礼貌
- 避免过于私密或不适合公开的内容
- 注意其他成员也能看到回复
- 保持适当的距离感

### 2. 群聊上下文管理 ✅

#### 共享消息缓存机制 🆕
- **所有bot共享同一个群聊消息缓存**
- 芊芊能看到婉清说了什么，婉清也能看到芊芊说了什么
- 主人的消息和所有bot的回复都会被记录
- 使用单例模式确保全局唯一的缓存实例
- **按chatId完全隔离**：不同群聊的消息互不干扰

#### 多群隔离机制 🔒
- 每个群聊有唯一的 `chatId`（Telegram群ID或Discord频道ID）
- 缓存结构：`Map<chatId, messages[]>`
- **示例**：
  - 古代群（芊芊、婉清）：chatId = `-1001234567890`
  - 现代群（曦月）：chatId = `-1009876543210`
  - 两个群的消息完全独立，互不可见
- 曦月在现代群中，完全看不到古代群的对话
- 芊芊和婉清在古代群中，完全看不到现代群的对话

#### 消息收集机制
- 持续收集群聊中的所有消息（包括主人和其他bot）
- 最多保留10条消息作为上下文
- 超过10条时，自动删除最早的消息
- **不再在回复后清空缓存**，而是持续累积

#### 时间过滤机制
- 默认只保留30分钟内的消息
- 可在配置文件中调整 `group_chat.message_expiry_minutes`
- 过期消息会被自动过滤，不会传递给AI

### 3. 群聊上下文格式

AI收到的群聊上下文格式如下：

```
[群聊上下文]
群成员：张三、李四、王五
最近的聊天记录：
张三: 今天天气真好
李四: 是啊，要不要出去玩
王五: 我也想去
[/群聊上下文]

当前用户消息内容
```

### 4. 数据库存储 ✅

**重要改动**：群聊消息现在会将上下文一起存入数据库

存储格式：
```
[群聊上下文]
群成员：张三、李四、王五
最近的聊天记录：
张三: 今天天气真好
李四: 是啊，要不要出去玩
王五: @bot 你觉得呢？

[/群聊上下文]

@bot 你觉得呢？
```

这样做的好处：
- ✅ Web界面可以看到完整的群聊对话
- ✅ 可以回顾当时的聊天场景
- ✅ 不会重复存储（因为有缓存清除机制）
- ✅ 每次bot回复后清空缓存，不会反复存入相同内容

数据库字段：
- `sender_name`: 发言人名字（触发回复的人）
- `user_id`: 格式为 `userId@chatId`（区分不同群聊）
- `content`: 完整消息内容（包含群聊上下文）
- `created_at`: 消息时间

### 5. Web界面显示 ✅

在web管理界面中：
- 群聊消息会显示发言人名字（蓝色标签）
- 显示完整的群聊上下文（包括其他人的对话）
- 格式：
  ```
  时间 [发言人]
  [群聊上下文]
  群成员：...
  最近的聊天记录：
  ...
  [/群聊上下文]
  
  实际消息内容
  ```
- 私聊消息不显示发言人，也没有群聊上下文

## 配置说明

在 `config.toml` 中可以配置：

```toml
[group_chat]
use_ai_judgment = true                    # 使用AI判断是否回复
question_response_probability = 0.6       # 问题回复概率
normal_response_probability = 0.2         # 普通消息回复概率
message_expiry_minutes = 30               # 消息过期时间（分钟）
```

## 测试方法

### 测试私聊场景
1. 在Telegram/Discord中私聊bot
2. 观察AI的回复是否更加亲密自然

### 测试群聊场景
1. 在群聊中@bot或回复bot的消息
2. 观察AI的回复是否保持得体
3. 发送多条消息后，AI应该能看到最近的聊天记录

### 测试Web界面显示
1. 在群聊中进行对话并触发bot回复
2. 打开web管理界面
3. 查看对话管理页面
4. 应该能看到完整的群聊上下文（包括其他人的对话）

### 测试消息过期
1. 在群聊中发送消息
2. 等待超过30分钟（或配置的时间）
3. 再次触发bot回复
4. 旧消息应该不会出现在上下文中

### 测试多群隔离
1. 创建两个不同的群聊
2. 在群聊A中添加芊芊和婉清
3. 在群聊B中添加曦月
4. 在群聊A中进行对话，触发芊芊回复
5. 在群聊B中@曦月
6. 曦月应该看不到群聊A的任何消息
7. 在web界面查看，两个群的对话历史应该完全独立

## 工作流程

```
群聊消息流程：
1. 用户A（主人）发送消息 → 存入共享缓存
2. Bot芊芊回复 → 存入共享缓存
3. Bot婉清回复 → 存入共享缓存
4. 用户A再次发送消息 → 存入共享缓存
5. Bot曦月被@触发 → 读取共享缓存（能看到主人和其他bot的对话）
6. 构建上下文 → 发送给AI
7. AI回复 → 存入共享缓存
8. 继续累积消息...（最多10条，30分钟内）
```

## 注意事项

1. ✅ 群聊上下文会存入数据库，方便在web界面查看
2. ✅ 使用共享缓存，所有bot能看到彼此的消息
3. ✅ 时间过滤确保不会存入过期消息
4. ✅ 每个群聊有独立的消息缓存
5. ✅ 存储的是触发回复时的完整场景，包括主人和其他bot的对话
6. ✅ 缓存持续累积，不会在回复后清空（只在超过10条或过期时清理）
