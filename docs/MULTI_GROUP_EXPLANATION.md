# 多群聊隔离说明

## 架构设计

共享缓存使用 `Map<chatId, messages[]>` 结构，每个群聊有独立的消息列表。

## 示例场景

### 群聊A：古代小丫环群（chatId: -1001234567890）
成员：
- 主人（莫非）
- 芊芊（古代小丫环）
- 婉清（古代小丫环）

对话：
```
莫非: 芊芊，你准备三把伞
芊芊: 好的主人，我这就去准备
莫非: 婉清，你跟厨师沟通一下
婉清: 明白了，我马上去安排
```

缓存内容：
```javascript
sharedGroupCache.recentGroupMessages.get('-1001234567890') = [
  { sender: '莫非', content: '芊芊，你准备三把伞', ... },
  { sender: '芊芊', content: '好的主人，我这就去准备', ... },
  { sender: '莫非', content: '婉清，你跟厨师沟通一下', ... },
  { sender: '婉清', content: '明白了，我马上去安排', ... }
]
```

成员：

```javascript
sharedGroupCache.recentGroupMessages.get('-1009876543210') = [

```

## 隔离机制

### 1. 按chatId完全隔离
- 每个群聊有唯一的 `chatId`
- 不同群聊的消息存储在不同的缓存条目中
- **曦月在群聊B中，完全看不到群聊A的消息**
- **芊芊和婉清在群聊A中，完全看不到群聊B的消息**

### 2. 数据库存储也是隔离的
- 每个群聊的 `user_id` 格式：`userId@chatId`
- 例如：
  - 群聊A：`user_id = "123456@-1001234567890"`
  - 群聊B：`user_id = "123456@-1009876543210"`
- 不同群聊的对话历史完全独立

### 3. 代码实现验证

```typescript
// 获取消息时，必须提供chatId
sharedGroupCache.getMessages(chatId)

// 存储消息时，也必须提供chatId
sharedGroupCache.addMessage(chatId, message)

// 不同的chatId = 不同的消息列表 = 完全隔离
```

## 工作流程示例

### 场景：曦月不在古代群中

1. **古代群（chatId: -1001234567890）**
   ```
   莫非: 芊芊，准备伞
   芊芊: 好的主人
   婉清: 我去安排午饭
   ```
   
   缓存状态：
   ```
   chatId: -1001234567890 → [莫非的消息, 芊芊的回复, 婉清的回复]
   ```

2. **现代群（chatId: -1009876543210）**
   ```
   莫非: 曦月，明天的课
   曦月: 已经安排好了
   ```
   
   缓存状态：
   ```
   chatId: -1009876543210 → [莫非的消息, 曦月的回复]
   ```

3. **曦月读取上下文时**
   ```typescript
   // 曦月在现代群中被@
   const messages = sharedGroupCache.getMessages('-1009876543210');
   // 只能看到现代群的消息，看不到古代群的消息
   ```

4. **芊芊读取上下文时**
   ```typescript
   // 芊芊在古代群中被@
   const messages = sharedGroupCache.getMessages('-1001234567890');
   // 只能看到古代群的消息，看不到现代群的消息
   ```

## 总结

✅ **完全隔离**：不同群聊的消息缓存是完全独立的
✅ **按需访问**：每个bot只能看到它所在群聊的消息
✅ **不会污染**：曦月不会被古代群的消息污染
✅ **灵活组合**：可以自由组合不同角色到不同群聊

这就是为什么使用 `chatId` 作为key的原因 - 它确保了不同群聊之间的完全隔离！
